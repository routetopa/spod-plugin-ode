Quick Introduction

Social features

An unify login allows an user to connect both SPOD or TET, whose aims is to enrich his/her profile so that s/he can search and receive resource on the platform based on his/her profile (to be considered for beta version of SPOD). Social Platform for Open Data (SPOD) is a social network where a first home page, e.g. Open Wall, after the authentication, shows an initial presentation of the social platform is displayed in the home page and interactive navigation of datasets of associated providers, showing the number of datasets and some summary for each dataset/provider.

SPOD offers each user a friendly environment right where s/he can easily navigation through several pages:

Whats new page, that it is constantly updating list of post or others activities in the Newsfeed section, including status updates, photo, open data visualisation and likes from users and can switch only to the My friends activities page; further it shows all users both latest and online

         My friends activities shows whats going on with a specific group of your friends.

         Members shows list of all online, latest users

On bottom of the page in SPOD, videos in YouTube or manual on line can facilitate and discover all features in the platform.

In addition to being a social network, SPOD lets you create and customize charts to Open Data datasets and share them for the discussions over open data, by forming or joining communities in SPOD with common interest and discuss common issues of relevance to local policy, service delivery and regulation.

SPOD makes easy all end-users , like citizens or public administrators, understanding of the available dataset, with a list provided from SPOD or remote dataset (via API made available by the Open Data provider). A wizard helps the user what type of visualisation to use. Before sharing the chosen chart, a preview of the chart is displayed and at the end, the user shares the visualisation to start the discussion.

The Agora shows the following characteristics:

         Level of discussion: it means that a Public Room can have a wide discussion (defined as a discussion where many datasets are used) or not, and can have a deep discussion (i.e. one that has many comments) or not.

2.3.5      Visualisation of the number of comments

Every Public Room is a tile and its dimension (width and height) reflects how deep is the discussion, i.e. the number of comments. The tiles that are in the top half, are given size 2 (in width and height), the other ones are given size 1. So, the available sizes are 1x1, 1x2, 2x1, 2x2.

2.3.6      Visualisation of the number of datasets used

Every Public Room is a tile and its dimension (width and height) reflects how wide is the discussion, i.e. the number of dataset in the discussion. The tiles that are in the top half, are given size 2 (in width and height), the other ones are given size 1. So, the available sizes are 1x1, 1x2, 2x1, 2x2.

2.3.7      Visualisation of the number of view

The tile color is indicative of the popularity, i.e. the number of views. We create a gradient from blue to red: the hotter is the color the larger is its popularity (hot). We decided that full red (the hottest color) correspond to the public room with the highest number of views, blue (the coldest color) with the public room with the minimum number of views. The tile color is calculated in percentage respect to the maximum value (100:maxValue=x:publicRoomViews).

2.3.8      Visualisation of the recency of activities

The order in which the rooms are shown (left to right, top-down) indicates which is the most recent commented room.

Figure 13 Room attributes: full red (the hottest color) correspond to the Public Room with the highest number of views, blue (the coldest color) with the Public Room with the minimum number of views.
